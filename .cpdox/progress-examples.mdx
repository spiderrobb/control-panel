# Visual Progress Examples

This page demonstrates the different progress states for running tasks.

## Progress States

### First Run - Animated Stripes
When you run a task for the first time, you'll see diagonal stripes animating across the background at 17% opacity. This indicates the task is running but we don't have historical data yet.

Try it: <TaskLink label="build:extension" />

### Subsequent Runs - Progress Bar
After the first run, the extension remembers how long the task took. On subsequent runs, you'll see a blue progress bar that fills from left to right based on the 10-run rolling average.

Try it: <TaskLink label="build:webview" />

### Long Running Tasks - Solid Blue
For tasks that run longer than 60 seconds, the progress indicator switches to a solid blue background to indicate steady-state execution.

Try it: <TaskLink label="test:integration" />

## Technical Details

The progress calculation uses a simple formula:

```
progress = (currentDuration / averageDuration) Ã— 100
```

The extension stores the last 10 execution durations for each task to calculate the rolling average. This data persists across VS Code sessions using the extension's global state.
